# 🏢 Space Booking — Бронирование помещений

Комплексное решение для управления резервированием пространства: переговорные комнаты, коворкинги, учебные классы, конференц-залы и другие типы помещений. Модуль демонстрирует продвинутые возможности Odoo: систему прав доступа, автоматизацию с помощью Cron, интеграцию с календарём, бизнес-валидацию, unit-тестирование и улучшенный UI с использованием HTML/CSS.

**Статус:** ✅ Завершён  
**Версия:** 1.0  
**Odoo:** 19.0  
**Лицензия:** LGPL-3

## ✨ Ключевые возможности

### 🗺️ Управление локациями
- Организация помещений по зданиям, этажам или секциям
- Адрес местоположения
- Автоматический подсчёт количества помещений в локации
- Быстрый просмотр всех помещений локации

### 🚪 Управление помещениями
- Полная информация о помещении:
  - Название
  - Тип (переговорная, коворкинг, учебная комната, конференц-зал, зал для тренировок, помещение для мероприятий, актовый зал, другое)
  - Вместимость (количество человек)
  - Местоположение
  - Доступное оборудование
- **Автоматический подсчёт активных бронирований**
- Возможность архивирования (active = False)
- Валидация вместимости (не менее 1)

### 📅 Бронирование помещений
- Создание бронирований с указанием:
  - Цели бронирования
  - Помещения
  - Времени начала и окончания
  - Организатора (по умолчанию текущий пользователь)
  - Заметок/комментариев
  - Напоминаний (поддержка стандартных напоминаний календаря)
- Автоматический расчёт длительности в часах
- Workflow с 4 статусами:
  - 📝 **Draft** (Черновик)
  - ✅ **Active** (Активно)
  - 🏁 **Done** (Завершено)
  - ❌ **Cancelled** (Отменено)

### 📆 Интеграция с календарём Odoo
- **При подтверждении бронирования автоматически создаётся событие в календаре**.
- Событие содержит название, время, участников (организатор) и напоминания.
- **При изменении времени бронирования** – событие автоматически обновляется.
- **При отмене или завершении бронирования** – событие удаляется.
- Напоминания настраиваются через поле `alarm_ids` (по умолчанию – за 15 минут).
- Календарный виджет в Odoo позволяет просматривать все бронирования в удобном формате (день, неделя, месяц) с возможностью Drag&Drop и быстрого создания.

### 🔒 Система безопасности
- **Две роли пользователей:**
  - 👤 **User** — обычный пользователь
  - 👑 **Manager** — менеджер бронирований

- **Права пользователя (User):**
  - ✅ Создавать свои бронирования
  - ✅ Редактировать свои бронирования
  - ✅ Отменять свои бронирования
  - ✅ Просматривать все бронирования (только чтение)
  - ❌ НЕ может удалять бронирования
  - ❌ НЕ может редактировать чужие бронирования
  - ❌ НЕ может управлять помещениями/локациями

- **Права менеджера (Manager):**
  - ✅ Все права пользователя +
  - ✅ Полный доступ ко всем бронированиям (CRUD)
  - ✅ Управлять помещениями и локациями
  - ✅ Завершать встречи
  - ✅ Возвращать отменённые в черновик

### 🛡️ Бизнес-логика и валидация

**Автоматические проверки:**
- Время окончания должно быть позже времени начала.
- Минимальная длительность – 10 минут, максимальная – 24 часа.
- Бронирование возможно не ранее чем через 10 минут от текущего времени.
- **Проверка пересечений:** нельзя подтвердить активное бронирование, если на это время уже есть другое активное бронирование в том же помещении (черновики не проверяются).
- При изменении даты события в календаре, связанного с бронированием, выбрасывается ошибка – изменение возможно только через бронирование.
- Автоматическое заполнение организатора (текущий пользователь) при создании.

### ⏰ Автоматизация (Cron)
- **Автоматическое завершение прошедших встреч** – задача выполняется каждые 10 минут, переводит статус активных бронирований с `end_time` меньше текущего времени в `done`.

### 🧪 Тестирование

**Полное покрытие тестами (4 тестовых файла):**

1. **Тесты моделей** (`test_space.py`):
   - Создание помещения
   - Валидация вместимости (не менее 1)
   - Подсчёт общего количества бронирований
   - Отображение имени (display_name)
   - Архивирование помещения

2. **Тесты бронирований** (`test_space_booking.py`):
   - Создание бронирования
   - Расчёт длительности
   - Валидация дат (конец после начала)
   - Обнаружение конфликтов (пересечений)
   - Переходы между статусами (workflow)
   - Отмена бронирования
   - Отображение имени

3. **Тесты ограничений** (`test_constraints.py`):
   - Проверка пересечений в одном помещении
   - Разрешение пересечений в разных помещениях
   - Исключение черновиков из проверки
   - Разрешение соседних (непересекающихся) бронирований

4. **Тесты безопасности** (`test_security.py`):
   - Пользователь может создать своё бронирование
   - Пользователь может читать своё бронирование
   - Менеджер видит все бронирования
   - Пользователь видит все помещения (чтение)
   - Пользователь не может удалить бронирование
   - Менеджер может удалить бронирование

**Запуск тестов:**
```bash
docker-compose run --rm web odoo \
    -d odoo \
    --db_host db \
    --db_user odoo \
    --db_password odoo \
    --test-enable \
    --stop-after-init \
    -u space_booking
```

## 🗂️ Модели данных

### 1. `space.location` — Местоположения

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | Char | Название местоположения (здание, этаж) |
| `address` | Text | Физический адрес |
| `space_ids` | One2many | Помещения в этом месте  |
| `space_count` | Integer | Количество помещений (вычисляемое) |

---

### 2. `space ` — Помещения

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | Char | Название помещения |
| `capacity` | Integer | Вместимость |
| `equipment` | Text | Доступное оборудование |
| `active` | Text | Описание оборудования |
| `space_type` | Selection | Тип помещения (переговорная,коворкинг, ...) |
| `location_id` | Many2one | Местоположение |
| `booking_ids` | One2many | Бронирования этого помещения |
| `booking_count` | Integer | Общее количество бронирований (вычисляемое) |
| `active_bookings_count` | Integer | Количество активных бронирований (вычисляемое) |
| `display_name` | Char | Отображаемое имя (вычисляемое) |

---

### 3. `space.booking` — Бронирования

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | Char | Цель встречи |
| `space_id` | Many2one | Помещение  |
| `partner_id` | Many2one | Организатор |
| `start_time` | Datetime | Время начала |
| `end_time` | Datetime | Время окончания |
| `duration` | Float | Длительность в часах |
| `notes` | Text | Заметки/повестка дня |
| `state` | Selection | Статус бронирования |
| `calendar_event_id` | Many2one | Событие в календаре (связь) |
| `alarm_ids` | Many2many | Напоминания (календарные) |
| `display_name` | Char | Отображаемое имя (вычисляемое) |

---

### 4. `res.partner ` — Контакты

| Поле | Тип | Описание |
|------|-----|----------|
| `booking_ids` | One2many | Бронирования пользователя |
| `booking_count` | Integer | Количество бронирований (вычисляемое) |
| `total_bookings_hours_month` | Float | Часы бронирования за текущий месяц (только active/done) |
| `total_bookings_hours_week` | Float | Часы бронирования за текущую неделю (только active/done) |

---

**Статусы:**
- `draft` — Черновик
- `active` — Активно
- `done` — Завершено
- `cancelled` — Отменено

## 🚀 Установка

### Предварительные требования
- Odoo 19.0
- PostgreSQL 15
- Python 3.10+

### Шаги установки

1. **Запустите Odoo:**
```bash
docker-compose up -d
```

2. **Создать БД**
   - Выбрать русский язык

3. **Настройка времени**
   - Перейдите в **Настроки - Пользователи** 
   - Выбрать пользователя 
   - Выбрать часовой пояс

4. **Обновите список приложений:**
   - Перейдите в **Приложения**
   - Нажмите **Обновить список приложений**

4. **Установите модуль:**
   - Найдите "Бронирование помещений"
   - Нажмите **Активировать**

5. **Настройте права пользователей:**
   - Перейдите в **Настрокйи → Пользователи**
   - Выберите пользователя
   - В разделе "Бронирование" выберите роль:
     - **No** — для обычных сотрудников
     - **Manager** — для администраторов
