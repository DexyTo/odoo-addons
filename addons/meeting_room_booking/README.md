# 🏢 Meeting Room Booking — Бронирование переговорных комнат

Продвинутый модуль для управления бронированием переговорных комнат с акцентом на безопасность, автоматизацию и улучшенный пользовательский интерфейс.

---

## 📋 Описание

Система бронирования переговорных комнат предназначена для управления расписанием встреч в офисе. Модуль демонстрирует продвинутые возможности Odoo: систему прав доступа, автоматизацию с помощью Cron, проверку бизнес-логики, unit-тестирование и улучшение UI с помощью HTML/CSS.

**Статус:** ✅ Завершён  
**Версия:** 1.0  
**Odoo:** 19.0  
**Лицензия:** LGPL-3

## ✨ Ключевые возможности

### 🗺️ Управление локациями
- Организация комнат по этажам и секциям
- Подсчёт количества комнат в локации
- Просмотр всех комнат локации одним кликом

### 🚪 Управление комнатами
- Полная информация о комнате:
  - Название
  - Вместимость (количество мест)
  - Расположение (этаж/секция)
  - Оборудование
- **Автоматический подсчёт активных бронирований**
- Возможность архивирования неиспользуемых комнат
- Валидация вводимых данных

### 📅 Бронирование комнат
- Создание бронирований с указанием:
  - Названия встречи
  - Комнаты
  - Времени начала и окончания
  - Организатора
  - Заметок/повестки дня
- Автоматический расчёт длительности встречи
- Workflow с 4 статусами:
  - 📝 **Draft** (Черновик)
  - ✅ **Active** (Активно)
  - 🏁 **Done** (Завершено)
  - ❌ **Cancelled** (Отменено)

### 📆 Календарное представление
- **Визуализация бронирований в календаре**
- Удобный просмотр расписания всех комнат
- **Интерактивное управление:**
  - Перетаскивание встреч (Drag & Drop)
  - Изменение длительности встречи прямо в календаре
  - Быстрое создание новых бронирований кликом на календаре
- Переключение режимов просмотра:
  - День
  - Неделя
  - Месяц
- Фильтрация

### 🔒 Система безопасности
- **Две роли пользователей:**
  - 👤 **User** — обычный пользователь
  - 👑 **Manager** — менеджер переговорных

- **Права пользователя (User):**
  - ✅ Создавать свои бронирования
  - ✅ Редактировать свои бронирования
  - ✅ Отменять свои бронирования
  - ✅ Просматривать все бронирования
  - ❌ НЕ может удалять бронирования
  - ❌ НЕ может редактировать чужие бронирования
  - ❌ НЕ может управлять комнатами/локациями

- **Права менеджера (Manager):**
  - ✅ Все права пользователя +
  - ✅ Управлять всеми бронированиями
  - ✅ Удалять бронирования
  - ✅ Создавать/редактировать комнаты
  - ✅ Управлять локациями
  - ✅ Завершать встречи
  - ✅ Возвращать отменённые в черновик

### 🛡️ Бизнес-логика и валидация

**Автоматические проверки:**
- ✅ Время окончания должно быть позже времени начала
- ✅ Нельзя забронировать комнату на прошедшее время
- ✅ Проверка пересечений: нельзя забронировать занятую комнату
- ✅ Автоматическое заполнение организатора (текущий пользователь)

### ⏰ Автоматизация (Cron)
- **Автоматическое завершение прошедших встреч**
- Переводит статус из `active` в `done`

### 🧪 Тестирование

**Полное покрытие тестами (30 тестов):**

1. **Тесты моделей** (`test_meeting_room.py`):
   - Создание комнаты
   - Валидация вместимости
   - Подсчёт активных бронирований
   - Архивирование комнат
   - Создание локации
   - Вычисление названия локации
   - Подсчёт комнат в локации

2. **Тесты бронирований** (`test_room_booking.py`):
   - Создание бронирования
   - Расчёт длительности
   - Валидация дат (конец после начала)
   - Переходы между статусами (workflow)
   - Копирование бронирований
   - Автозаполнение организатора

3. **Тесты ограничений** (`test_constraints.py`):
   - Проверка пересечений в одной комнате
   - Разрешение пересечений в разных комнатах
   - Исключение черновиков из проверки
   - Разрешение смежных бронирований

4. **Тесты безопасности** (`test_security.py`):
   - Пользователь может создать своё бронирование
   - Пользователь может редактировать своё
   - Пользователь НЕ может редактировать чужое
   - Пользователь НЕ может удалять
   - Пользователь может отменить своё
   - Пользователь НЕ может отменить чужое
   - Менеджер может редактировать любое
   - Менеджер может удалять
   - Пользователь видит все бронирования (чтение)
   - Пользователь НЕ может управлять комнатами
   - Менеджер может управлять комнатами
   - Пользователь НЕ может управлять локациями
   - Менеджер может управлять локациями

## 🗂️ Модели данных

### 1. `meeting.room.location` — Локации

| Поле | Тип | Описание |
|------|-----|----------|
| `floor` | Integer | Номер этажа |
| `section` | Char | Секция (A, B, C...) |
| `name` | Char | Полное название  |
| `room_ids` | One2many | Комнаты в этой локации |
| `room_count` | Integer | Количество комнат |

---

### 2. `meeting.room` — Переговорные комнаты

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | Char | Название комнаты |
| `capacity` | Integer | Вместимость |
| `location_id` | Many2one | Локация комнаты |
| `equipment` | Text | Описание оборудования |
| `active` | Boolean | Активна ли комната |
| `booking_ids` | One2many | Все бронирования комнаты |
| `active_bookings_count` | Integer | Активные бронирования |

---

### 3. `room.booking` — Бронирования

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | Char | Цель встречи |
| `room_id` | Many2one | Комната  |
| `partner_id` | Many2one | Организатор |
| `start_time` | Datetime | Время начала |
| `end_time` | Datetime | Время окончания |
| `duration` | Float | Длительность в часах |
| `notes` | Text | Заметки/повестка дня |
| `state` | Selection | Статус бронирования |

**Статусы:**
- `draft` — Черновик
- `active` — Активно
- `done` — Завершено
- `cancelled` — Отменено

---

### 4. `res.partner` (расширение) — Контакты

**Добавленные поля:**

| Поле | Тип | Описание |
|------|-----|----------|
| `total_bookings_hours_month` | Float | Часы бронирования за месяц |
| `total_bookings_hours_week` | Float | Часы бронирования за неделю |
| `booking_ids` | One2many | Мои бронирования |

## 🚀 Установка

### Предварительные требования
- Odoo 19.0
- PostgreSQL 13+
- Python 3.10+

### Шаги установки

1. **Запустите Odoo:**
```bash
docker-compose up -d
```

2. **Обновите список приложений:**
   - Перейдите в **Apps** (Приложения)
   - Нажмите **Update Apps List** (Обновить список приложений)

4. **Установите модуль:**
   - Найдите "Бронирование переговорных"
   - Нажмите **Install** (Активировать)

5. **Настройте права пользователей:**
   - Перейдите в **Settings → Users & Companies → Users**
   - Выберите пользователя
   - В разделе "Бронирование" выберите роль:
     - **No** — для обычных сотрудников
     - **Manager** — для администраторов
